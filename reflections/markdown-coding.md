# Reflection: Coding in Markdown

## STAR Method Reflection

### Situation
As part of my Work Integrated Learning (WIL) module at the Cape Peninsula University of Technology,
I was required to build a digital portfolio hosted on GitHub. The portfolio needed to present my CV,
projects, and reflections in a structured, readable format. I had not previously written content
using Markdown and was unfamiliar with its syntax and conventions.

### Task
My task was to learn and apply GitHub Flavoured Markdown (GFM) to code my CV and structure my
portfolio repository. This included using headings, lists, tables, links, and embedded media — all
using plain-text Markdown syntax rather than a visual editor or HTML.

### Action
I began by reading the official GitHub Markdown documentation to understand the basic syntax.
I then practised formatting my CV by converting sections such as personal details, education,
skills, and projects into structured Markdown. I used headings to create a clear hierarchy,
tables for skills and qualifications, and bullet points for experience items. I also learned
how to embed links and images, and how to preview my Markdown output directly on GitHub before
finalising the content. When I encountered formatting issues. such as tables not rendering
correctly. I researched the cause and corrected the syntax.

### Result
The result was a fully coded CV in Markdown that renders cleanly on both GitHub and GitHub Pages.
Through this process I developed a practical understanding of Markdown as a developer communication
tool. I now appreciate why Markdown is widely used in README files, documentation, and pull requests. It is lightweight, readable in raw form, and renders consistently across platforms. This is a
skill I will continue to use throughout my development career.
