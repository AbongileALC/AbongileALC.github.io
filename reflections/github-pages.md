# Reflection: Publishing on GitHub Pages

## STAR Method Reflection

### Situation
After building my digital portfolio as a GitHub repository, I was required to deploy it as a
live website using GitHub Pages. While I was familiar with GitHub for version control, I had
not previously used GitHub Pages to publish a live site from a repository. The portfolio needed
to be publicly accessible via a URL for submission and assessment.

### Task
My task was to configure my GitHub repository to deploy correctly through GitHub Pages, ensure
all pages and links rendered as expected on the live site, and submit the published URL as my
final portfolio submission via Blackboard.

### Action
I navigated to the Settings of my GitHub repository and enabled GitHub Pages by selecting the
main branch as the publishing source. I verified that my repository was named correctly following
the `username.github.io` convention to ensure it deployed to the root domain. After enabling
Pages, I visited the live URL to check that all Markdown files were rendering correctly, that
navigation links between pages worked, and that embedded media such as my CV and video were
displaying as expected. Where I found issues such as relative links not resolving correctly.
I corrected the file paths and committed the changes, then re-checked the deployed site.

### Result
My portfolio was successfully published at `AbongileALC.github.io` and is accessible to anyone
with the link. This experience gave me practical exposure to static site deployment — a workflow
commonly used in professional development environments. I now understand how GitHub Pages works
as a hosting solution and how repository structure directly affects what gets rendered on the
live site. Deploying a real, publicly accessible portfolio also gave me a tangible professional
asset I can share with future employers.
